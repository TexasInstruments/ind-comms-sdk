Beckhoff Library creation

To create your own release / licensed version of the Beckhoff Slave SampleCode for Sitara usage you have to follow these steps.

Prerequisites

To create your licensed library we recommend the usage of a Linux Development host or at least WSL with e.g. Ubuntu image. Tools required on development host:

  • cmake (Version 3.20 or later)
  • dos2unix utility
  • patch utility
  • bash utility
  • unzip utility

For FreeRTOS

  • Code Composer Studio (currently 12.1 or later)
  • TI CLang (2.1.2 LTS)
  • MCU+SDK for your MCU (MCU+SDK 8.5.0.24 or later)
  • SysConfig 1.14

For FreeRTOS/Jacinto (AM65)

  • Code Composer Studio (currently 12.1 or later)
  • TI CLang (1.3.0 LTS)
  • CoreSDK for your MCU (SDK 8.2.0.05 or later)

For TIRtos

  • Code Composer Studio (currently 12.1 or later)
  • gcc 10
  • TI cgt arm (20.2.5)
  • PDK (latest revision)

For Linux

  • gcc 10
  • arago SDK (8.x)

BuildSteps

  1. Obtain Slave Sample Code from ETG/Beckhoff (Vendor ID and ETG membership required, free of charge in that case) Current version support: 5i13
  2. Unzip the V5i13 package
  3. Copy SlaveFiles.zip from that package to your development host
  4. Create a folder for library creation (e.g. sscCustomLibrary)
  5. unzip the code package for library creation into folder (e.g. sscCustomLibrary, example shown for AM243x which is equipped on TI Launchpad): 
     cd sscCustomLibrary
 unzip ~/Downloads/ethercat_slave_AM243x_r5f_freertos_customerSlaveSampleCode.zip
  6. enter the created folder: 
     cd  ethercat_slave_AM243x_r5f_freertos_customerSlaveSampleCode/
  7. create a folder for your build: 
     mkdir build
  8. enter the created build folder: 
     cd build
  9. call the convenient build script with the location of the Beckhoff SlaveFiles.zip archive: 
     ../build.sh ~/Downloads/SSC_V5i12/SlaveFiles.zip
  10. The libray is create in your build folder and goes by the name libbkhfSsc.a
  11. Copy the library to your EtherCAT_Slave project folder and link it instead of the given sample library

Changes on Bechoff SSC definitions

The Beckhoff SSC library is parametrized using defines. These defines are located in the ecat_def.h file. Apply the patch and if there is any define to set differently, modify it and then build the library again. For example, the default MAX_MBX_SIZE is set to 0x0400 but it is possible to adjust it manually.
