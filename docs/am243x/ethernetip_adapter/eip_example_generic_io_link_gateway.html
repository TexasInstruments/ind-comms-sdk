Scope

This section discusses an example application that demonstrates a vendor specific implementation of an EtherNet/IP to IO-Link gateway, using a value of 0x2B (Generic Device) as Device Type (Attribute 2 in the Identity Object Instance 1).

Partially objects as specified in the Object Library of Volume 7C Integration of IO-Link Devices into the CIP Architecture [3] are reused. They are complemented by vendor specific objects to enable explicit messaging access to parameters of connected IO-Link devices.

Object Library

IO-Link Master PHY Object

Class Code: 0x10C

The IO-Link Master PHY Object represents all physical properties of IO-Link within the IO-Link Master. Each instance of the IO-Link Master PHY Object represents a channel on the IO-Link Master.
